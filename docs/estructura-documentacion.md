# Estructura sugerida para la Documentación

## Portada

- Nombre de la institución
- Nombre del proyecto
- Integrantes
- Docentes
- Año
- Curso

## 1 - Índice

## 2 - Abstract

## 3 - Introducción

Aquí explican:

- Qué es el proyecto
- Por qué surge
- Qué necesidad intenta resolver
- Breve descripción del sistema/producto

Debe ser general y entendible para cualquier lector.

## 4 - Marco Contextual

Describe el contexto donde aparece el problema.

Ejemplos:

- Centro educativo
- Empresa
- Biblioteca
- Comercio
- Comunidad

Debe incluir:

- Situación actual
- Cómo trabajan actualmente
- Qué dificultades existen
- Quiénes son los usuarios

## 5 - Planteamiento del Problema

Aquí se define claramente el problema.

Debe responder:

- ¿Qué ocurre?
- ¿Por qué es un problema?
- ¿A quién afecta?
- ¿Qué consecuencias tiene?

Ejemplo: "El liceo no posee un sistema digital para registrar préstamos de materiales, lo que genera pérdida de información y demoras administrativas."

## 6 - Justificación

Explica:

- Por qué vale la pena hacer el proyecto
- Qué beneficios aporta
- Qué impacto tendrá

Puede incluir:

- Beneficios educativos
- Sociales
- Organizacionales
- Tecnológicos

## 7 - Objetivos

### Objetivo General

Qué pretende lograr el proyecto.

Ejemplo: "Desarrollar un sistema web para gestionar préstamos de materiales educativos."

### Objetivos Específicos

Acciones concretas.

Ejemplo:

- Analizar necesidades del usuario
- Diseñar interfaz
- Crear base de datos
- Implementar autenticación
- Realizar pruebas

## 8 - Marco Teórico

Aquí entran los conceptos teóricos.

Dependiendo de las materias:

- Ingeniería de software
- Programación
- Bases de datos
- Administración de S.O
- Ciberseguridad
- Metodologías ágiles

Ejemplos de temas:

- Qué es un sistema de información
- Ciclo de vida del software
- Bases de datos relacionales
- HTML/CSS/JS
- Ciberseguridad

## 9 - Metodología de Trabajo

Aquí explican:

- Cómo trabajaron
- Qué metodología usaron

Por ejemplo:

- Scrum
- Cascada
- Kanban

También:

- Roles del equipo
- Cronograma
- Herramientas utilizadas
- Organización de tareas

## 10 - Relevamiento y Análisis

Especificación de Requerimientos de Software

## 11 - Modelado del Sistema

### Casos de Uso

- Diagrama de casos de uso

### Diagramas de Flujo

- Procesos principales

### Modelo Entidad Relación (MER)

## 12 - Diseño de la Solución

### Diseño de interfaz

- Bocetos

### Arquitectura del sistema

- Cliente-servidor
- Web

### Diseño de Base de Datos

- Tablas - Relaciones (MR: Modelo Relacional)

## 13 - Desarrollo e Implementación

Aquí explican:

- Tecnologías utilizadas
- Lenguajes
- Frameworks
- Herramientas

Ejemplo:

- HTML
- CSS
- JavaScript
- Python
- MySQL

También:

- Capturas del sistema
- Explicación de módulos

## 14 - Pruebas del Sistema

- Pruebas funcionales
- Pruebas de caja negra
- Casos de prueba

Pueden usar tablas: Caso | Entrada | Resultado esperado

También:

- Validación con usuarios
- Corrección de errores

## 15 - Resultados

Qué lograron.

Por ejemplo:

- Sistema funcionando
- Automatización
- Mejora de tiempos
- Opinión de usuarios

## 16 - Conclusiones

Reflexión final:

- Qué aprendieron
- Qué salió bien
- Dificultades
- Posibles mejoras
- Resultado obtenido

## 17 - Recomendaciones

Qué se podría hacer a futuro.

Ejemplo:

- Aplicación móvil
- Integración con nube
- Nuevos módulos
