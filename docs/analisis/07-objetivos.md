# 7 - Objetivos

## Objetivo General

Desarrollar un sistema web modular para el Hospital de Clínicas que permita la gestión digital de documentación informativa para pacientes con acceso mediante código QR, y la trazabilidad de traslados en ambulancia a nivel nacional, integrado al sistema centralizado existente de la institución.

## Objetivos Específicos

### Módulo de Gestión de Documentación para Pacientes

1. **OE1** — Implementar un módulo de carga y administración de documentos informativos que permita al personal administrativo subir, editar y organizar archivos destinados a los pacientes del hospital.

2. **OE2** — Desarrollar un sistema de generación de códigos QR asociados a cada documento, permitiendo a los pacientes acceder al contenido digital mediante el escaneo desde sus dispositivos móviles.

3. **OE3** — Diseñar e implementar un subsistema de encuestas de satisfacción que permita a los pacientes completar formularios digitales y almacenar las respuestas en el servidor para su posterior análisis.

4. **OE4** — Reducir los costos operativos asociados a la impresión de documentos informativos mediante la digitalización del acceso a la documentación.

### Módulo de Trazabilidad de Ambulancias

5. **OE5** — Desarrollar un módulo de registro de solicitudes de traslado que permita almacenar y gestionar los datos de cada operación: conductor, copiloto, paciente/elemento, origen, destino, horarios y rutas.

6. **OE6** — Implementar un sistema de seguimiento en tiempo real del estado de cada traslado, desde la salida del hospital hasta el retorno, permitiendo al personal administrativo visualizar el ciclo operativo completo.

7. **OE7** — Desarrollar la funcionalidad de gestión de rutas dentro del circuito nacional, contemplando la planificación y el registro de los trayectos realizados.

### Generales

8. **OE8** — Integrar ambos módulos al panel principal del sistema centralizado del Hospital de Clínicas, utilizando las credenciales de acceso existentes (usuario y contraseña).

9. **OE9** — Diseñar e implementar una interfaz web responsive utilizando PHP, MySQL, HTML, CSS, JavaScript y Bootstrap, siguiendo una arquitectura hexagonal.

10. **OE10** — Realizar pruebas funcionales del sistema para garantizar la correcta operación de los módulos desarrollados.
