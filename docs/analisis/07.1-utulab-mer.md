# 7.1 - Modelo Entidad Relación (MER) — Base de Datos del Sistema

## Modelo Entidad Relación de Elyra

El sistema Elyra se compone de **14 tablas** organizadas en tres módulos: Identidad (4 tablas), Documentación (5 tablas) y Ambulancias (5 tablas).

### Módulo de Identidad (4 tablas)

```
USUARIO ← tabla base con datos comunes de toda persona
├── id (PK), nombre, apellido, email, documento_identidad, tipo, created_at

FUNCIONARIO ← extiende USUARIO, para empleados con login
├── id (PK/FK), username, password_hash, licencia, telefono, rol, activo

PACIENTE ← extiende USUARIO, para personas que acceden por QR
├── id (PK/FK), token_acceso, codigo_qr_id (FK → CODIGO_QR)

CODIGO_QR ← código QR que los pacientes escanean
├── id (PK), nombre, descripcion
```

### Módulo de Documentación (6 tablas)

```
CATEGORIA: id (PK), nombre, descripcion
ENCUESTA: id (PK), titulo, descripcion, activa, creada_por (FK), timestamps
PREGUNTA: id (PK), encuesta_id (FK), tipo, texto, opciones, requerida, orden
DOCUMENTO: id (PK), titulo, descripcion, archivo_path, codigo_qr_id (FK → CODIGO_QR), qr_path,
           categoria_id (FK), encuesta_id (FK), subido_por (FK), activo, timestamps
RESPUESTA: id (PK), sesion_token, encuesta_id (FK), pregunta_id (FK),
           token_paciente, valor_opcion, valor_texto, valor_numerico, created_at
```

### Módulo de Ambulancias (5 tablas)

```
VEHICULO: id (PK), patente, modelo, anio
RUTA: id (PK), nombre, origen, destino, distancia_km, descripcion
TRASLADO: id (PK), codigo, conductor_id (FK), copiloto_id (FK), vehiculo_id (FK),
          ruta_id (FK), origen, destino, estados_horarios, estado, registrado_por (FK)
ELEMENTO_TRASLADO: id (PK), traslado_id (FK), tipo, paciente_id (FK), descripcion, cantidad
HISTORIAL_ESTADO: id (PK), traslado_id (FK), estado_anterior, estado_nuevo,
                  actualizado_por (FK), created_at
```

### Resumen

| # | Tabla | Módulo | Descripción |
|---|-------|--------|-------------|
| 1 | USUARIO | Identidad | Datos comunes de toda persona |
| 2 | FUNCIONARIO | Identidad | Empleados con login (admin, superadmin, conductor) |
| 3 | PACIENTE | Identidad | Personas que acceden por QR o viajan en ambulancia |
| 4 | CODIGO_QR | Identidad | Códigos QR que los pacientes escanean para acceder a documentos |
| 5 | CATEGORIA | Documentación | Clasificación de documentos por área médica |
| 6 | ENCUESTA | Documentación | Formularios de satisfacción |
| 7 | PREGUNTA | Documentación | Cada pregunta de una encuesta |
| 8 | DOCUMENTO | Documentación | PDF informativos con su código QR |
| 9 | RESPUESTA | Documentación | Respuestas de pacientes agrupadas por sesión |
| 10 | VEHICULO | Ambulancias | Flota de ambulancias |
| 11 | RUTA | Ambulancias | Rutas del circuito nacional |
| 12 | TRASLADO | Ambulancias | Solicitudes de viaje en ambulancia |
| 13 | ELEMENTO_TRASLADO | Ambulancias | Qué se traslada (paciente, órgano, equipo o insumo) |
| 14 | HISTORIAL_ESTADO | Ambulancias | Bitácora de cambios de estado del viaje |

> El detalle completo con campos, tipos, índices y relaciones se encuentra en la sección 11.3 y en `diseno/03-diseno-base-datos.md`.
