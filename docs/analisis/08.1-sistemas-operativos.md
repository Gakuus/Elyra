## Justificación — Elecciones tecnológicas

### Debian 12 (Bookworm) — MV de servicios

Se eligió **Debian 12** como sistema operativo base para las máquinas virtuales del cluster Proxmox por las siguientes razones:

1. **Estabilidad probada**: Debian es conocido por su extrema estabilidad. Las actualizaciones son conservadoras y probadas exhaustivamente antes de liberarse, ideal para un entorno institucional donde la confiabilidad es crítica.

2. **Soporte LTS extendido**: Debian 12 cuenta con soporte hasta 2028, garantizando actualizaciones de seguridad durante todo el ciclo de vida del proyecto.

3. **Bajo consumo de recursos**: Debian funciona eficientemente incluso con recursos limitados, maximizando el rendimiento de cada máquina virtual.

4. **Entorno mínimo (CLI)**: Se instaló sin entorno de escritorio, únicamente con las funcionalidades requeridas, eliminando paquetes innecesarios como drivers de audio, lo que reduce la superficie de ataque y el consumo de recursos.

5. **Compatibilidad con Docker y LAMP**: Debian tiene soporte nativo tanto para contenedores Docker como para el stack LAMP utilizado por Elyra.

6. **Licencia libre**: Sin costos de licencia, alineado con los recursos institucionales.

7. **Seguridad**: Debian cuenta con un equipo de seguridad dedicado que responde rápidamente ante vulnerabilidades.

### Ubuntu Server 24.04 LTS — Proxy reverso en AWS

Se eligió **Ubuntu Server** para la instancia en la nube de AWS por las siguientes razones:

1. **Integración nativa con AWS**: Ubuntu es la distribución mejor soportada en EC2, con imágenes optimizadas y actualizaciones de kernel sin reinicio (Livepatch).

2. **Amplia comunidad y documentación**: Al ser una de las distribuciones más populares en la nube, cuenta con abundante documentación orientada a despliegues cloud.

3. **Soporte LTS hasta 2029**: Garantiza actualizaciones de seguridad durante todo el ciclo de vida del proyecto.

4. **Familiaridad técnica**: Base Debian/APT, lo que facilita la administración al mismo equipo que gestiona las MV con Debian.

### Proxmox VE — Hipervisor

Se eligió **Proxmox VE** como plataforma de virtualización por las siguientes razones:

1. **Virtualización tipo 1 (bare-metal)**: Al instalarse directamente sobre el hardware, ofrece mejor rendimiento y eficiencia que los hipervisores tipo 2.

2. **Alta disponibilidad**: Permite crear un cluster con múltiples nodos replicados, de modo que si un nodo falla el otro toma el control automáticamente (respaldo en caliente).

3. **Código abierto y sin costos de licencia**: Basado en Debian, con licencia GPL.

4. **Administración web integrada**: Proporciona una interfaz web para gestionar máquinas virtuales, contenedores, almacenamiento y redes.

5. **Soporte para contenedores LXC y KVM**: Permite ejecutar tanto máquinas virtuales completas (KVM) como contenedores ligeros (LXC).

## Arquitectura de infraestructura

La infraestructura del ITSP donde se despliega Elyra sigue el siguiente esquema:

### Capa 1 — Proxy reverso en AWS (Ubuntu Server + Nginx)

Se cuenta con una instancia en la nube de **AWS EC2** con **Ubuntu Server** y **Nginx** como proxy reverso. Esta máquina virtual posee la dirección IP pública y es la puerta de entrada a toda la plataforma. Su función principal es:

- Recibir las solicitudes HTTPS de todos los clientes externos.
- Actuar como **proxy reverso Nginx**, derivando todo el tráfico hacia los servidores internos del ITSP a través de una **VPN**.
- Proveer una capa adicional de seguridad al ocultar la infraestructura interna detrás de un único punto de entrada.

El proxy reverso permite que los servicios internos no estén expuestos directamente a internet, reduciendo significativamente la superficie de ataque.

### Capa 2 — Proxmox (ITSP)

En las instalaciones del ITSP se implementó **Proxmox VE** como hipervisor tipo 1 (bare-metal), basado en Debian GNU/Linux. Sobre este se aprovisionó una máquina virtual con **Debian 12**, configurada sin entorno de escritorio (CLI-only), con los paquetes mínimos necesarios y todos los servicios innecesarios desinstalados (drivers de audio, bluetooth, etc.).

### Capa 3 — VM Debian 12 (Servidor Flask)

Esta máquina virtual expone los siguientes componentes:

**Nginx (puerto 8080):** Actúa como enrutador interno de las solicitudes, derivando cada ruta al servicio correspondiente:

| Ruta | Destino | Descripción |
|------|---------|-------------|
| `/registro/*` | **Flask + Gunicorn** (puerto 5000) | Servicio de registro de usuarios (`flaskregistro.service`) |
| `/proyectos` | **dbitsp** (SSH Server Debian) | Alojamiento de proyectos de estudiantes con MySQL (`doc_users`) y directorios home (`/var/www/alumno/`) |
| `/websec-lab/` | **Docker websec-lab** | Entorno que simula una web vulnerable para prácticas de ciberseguridad |
| `/terminal/` | **ttyd** (puerto 7681) | Terminal web con autologin, basada en `nginx:alpine` |
| `/` | **Docker dos-target** | Entorno que simula una PC vulnerable para prácticas de seguridad |

La VM es accesible desde la LAN interna en la dirección `192.168.2.238:80`.

### Capa 4 — Administración remota

La totalidad de la infraestructura se administra de forma remota mediante **SSH** (Secure Shell). No se requiere acceso físico a los servidores para tareas de mantenimiento, actualización o monitoreo.

## Manual de Instalación — Debian 12 (Bookworm) para MV en Proxmox

### Requisitos del Sistema (MV)

| Componente | Mínimo | Recomendado |
|------------|--------|-------------|
| Procesador | 1 vCPU (64-bit) | 2 vCPU |
| RAM | 512 MB | 2 GB |
| Disco duro | 10 GB | 20 GB+ |
| Conexión de red | Bridge de Proxmox | Bridge de Proxmox |

### Paso 1: Descargar la ISO

1. Ir a https://www.debian.org/download
2. Descargar la imagen ISO de **Debian 12 (Bookworm)** para arquitectura **amd64**
   - Opción recomendada: **debian-12.x.x-amd64-netinst.iso** (liviana, requiere internet)

### Paso 2: Subir la ISO a Proxmox

1. En la interfaz web de Proxmox, seleccionar el nodo.
2. Ir a **local (storage) → ISO Images → Upload**.
3. Seleccionar el archivo ISO descargado y subirlo.

### Paso 3: Crear la máquina virtual

1. En Proxmox, hacer clic en **Create VM**.
2. Configurar:
   - **General**: Nombre de la MV (ej: `elyra-servicios`).
   - **OS**: Seleccionar la ISO de Debian 12 subida.
   - **System**: scsi, VirtIO.
   - **Disks**: 20 GB en storage local o compartido.
   - **CPU**: 2 cores.
   - **Memory**: 2048 MB.
   - **Network**: Bridge vmbr0, modelo VirtIO.

### Paso 4: Iniciar la instalación

1. Iniciar la MV desde la interfaz web de Proxmox (botón **Start**).
2. Abrir la consola (**Console**).
3. Seleccionar **"Install"**.
4. Idioma: **Español** (o **English**).
5. Ubicación: **Uruguay**.
6. Teclado: **Español (Latinoamericano)**.

### Paso 5: Configuración de red

1. El instalador detectará automáticamente la interfaz de red.
2. Si hay DHCP en la red del ITSP, la configuración será automática.
3. Si se requiere IP fija (recomendado):
   - Seleccionar "Configurar red manualmente"
   - Ingresar IP, máscara, gateway y DNS según la red interna del ITSP

### Paso 6: Nombre del equipo

- **Nombre del servidor**: según el rol (ej: `elyra-vm1`, `elyra-vm2`)
- **Dominio**: `itsp.edu.uy` (o el dominio institucional)

### Paso 7: Configuración de usuarios

1. **Contraseña de root**: Ingresar una contraseña segura.
   - Alternativa: Dejar vacío y usar `sudo` (recomendado).
2. **Nombre de usuario**: según el administrador responsable.
3. **Contraseña del usuario**: Contraseña segura de al menos 12 caracteres.

### Paso 8: Particionado

**Opción recomendada — Guiado - usar todo el disco:**

1. Seleccionar "Guiado - usar todo el disco".
2. Seleccionar el disco virtual.
3. Elegir **"Todos los archivos en una partición"** (esquema simple para MV).

### Paso 9: Selección de software

Para una instalación mínima (servidor sin escritorio):

```
Seleccionar solo:
- Servidor SSH
- Utilidades estándar del sistema
```

No instalar entorno de escritorio, servidor web ni base de datos desde el instalador; estos se instalarán manualmente según los requerimientos de cada VM.

### Paso 10: Finalizar instalación

1. Confirmar que GRUB se instale en el disco principal.
2. El sistema se reiniciará.
3. La MV quedará lista para ser administrada vía SSH.

### Paso 11: Post-instalación — Mínimo común

```bash
# Actualizar el sistema
sudo apt update && sudo apt upgrade -y

# Instalar utilidades básicas
sudo apt install -y git curl wget unzip

# Configurar firewall
sudo apt install -y ufw
sudo ufw allow OpenSSH
sudo ufw enable
```

### Paso 12: Instalación de Docker

Dado que varios servicios se ejecutan como contenedores, se instala Docker Engine:

```bash
# Instalar dependencias
sudo apt install -y ca-certificates curl gnupg

# Agregar repositorio oficial de Docker
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | \
    sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) \
    signed-by=/etc/apt/keyrings/docker.gpg] \
    https://download.docker.com/linux/debian \
    bookworm stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Instalar Docker
sudo apt update && sudo apt install -y docker-ce docker-ce-cli containerd.io

# Verificar instalación
sudo docker --version

# Habilitar Docker para iniciar automáticamente
sudo systemctl enable docker
```

### Paso 13: Configuración de servicios específicos

**VM1 — Plataforma Elyra + Docker:**

```bash
# Instalar stack LAMP
sudo apt install -y apache2 mariadb-server php php-mysql \
    php-cli php-common php-json php-mbstring php-xml \
    php-curl php-zip php-gd php-bcmath

# Habilitar servicios
sudo systemctl enable apache2
sudo systemctl enable mariadb

# Configurar firewall para servicios
sudo ufw allow 'Apache Full'
sudo ufw allow 3306/tcp   # MariaDB (solo red interna)
```

**VM2 — Servicios web para estudiantes:**

```bash
# Instalar servidor web
sudo apt install -y apache2

# Habilitar servicios
sudo systemctl enable apache2
```

### Solución de problemas comunes

| Problema | Causa probable | Solución |
|----------|---------------|----------|
| No arranca la MV | ISO no subida correctamente | Verificar integridad de la ISO en storage Proxmox |
| Sin conexión de red en la MV | Bridge mal configurado | Verificar vmbr0 en Proxmox y reiniciar red |
| Apache no inicia | Puerto 80 ocupado | `sudo netstat -tulpn \| grep :80` y liberar puerto |
| MySQL no arranca | Problema de permisos | `sudo chown -R mysql:mysql /var/lib/mysql` |
| Docker no funciona | CGroups v1 vs v2 | Verificar kernel: `uname -r` |
